Two different things, one workflow

Git vs GitHub

🔧 Git

The tool on your computer that tracks every change to your code. Works fully offline. Created by the same person who made Linux.

+

☁️ GitHub

A website that stores your Git projects online — so they're backed up, shareable, and visible to the world (and recruiters).

Analogy: Git is the camera that takes the photos. GitHub is the cloud album where you store and share them.

The core idea

A commit = a saved snapshot

Each commit freezes your whole project at one moment, with a short message saying what changed. The chain of commits is your project's history — your time machine.

Why bother learning this

What Git/GitHub buys you

⏪ Undo anything

Broke it? Jump back to any commit. No more _final2 files — ever.

☁️ Never lose work

Pushed to GitHub = backed up online. Laptop dies, your code lives.

👥 Work with others

Many people edit the same project without overwriting each other (Day 6).

💼 Get hired

Your GitHub is your portfolio. Recruiters look here before your resume.

The two places your code lives

Local repo ↔ Remote repo

Push = send your commits up to GitHub. Pull = bring others' changes down. (Hands-on with these on Day 5.)

The words people throw around

Core terminology — decoded

📁 Repository ("repo")

A project folder that Git is tracking — your code + its full history.

📸 Commit

One saved snapshot + a message describing the change.

⬆️ Push / ⬇️ Pull

Upload your commits to GitHub / download new ones from it.

🍴 Clone

Copy an online repo down to your computer to work on it.

Don't memorise — you'll use these starting tomorrow. Today just recognise them.

Step 0 — do this today

Create your GitHub account

  • Go to github.com → Sign up (it's free).
  • Pick a professional username — this becomes your URL: github.com/yourname. Recruiters see it. Avoid xX_coder_69_Xx.
  • Use an email you'll keep. Add a real photo later.
  • That's it — you now have a home for every project in this course.

Your 4 course projects + capstone will all live here. By Day 45 this page is your portfolio.

What you'll see online

Anatomy of a repo page

📄 The file list

Your folders & files, exactly as on your computer.

📖 The README

The README.md shows under the files — it's the project's front page (what it does, how to run it).

🕑 Commits

A clickable history of every change, by whom, when.

⭐ Stars / forks

How others bookmark & copy your project.

A good README is the difference between "some files" and "a project someone respects."
