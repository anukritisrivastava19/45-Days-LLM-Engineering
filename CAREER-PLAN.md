# SSAI-101 — Career Launchpad (Daily 30-min Track)

**Softpro School of AI — the "get hired / get clients" track that runs alongside the code.**

> The technical track (see [`COURSE-PLAN.md`](COURSE-PLAN.md)) makes students *able*. This track
> makes them *visible, specialized, and hireable*. It runs **every day for 30 minutes** over a
> presentation, for all 45 days.

## The thesis (why this track exists)
Most students believe: **learn skills → get job.** Reality: skills are table stakes. The people who
actually get hired (or land freelance clients) win on **two extra levers**:

1. **Content / building in public** — a visible body of work that makes opportunities come to *you*.
2. **Specialization** — being known for *one specific thing* beats being a vague generalist.

This is how the trainer got most of their own jobs and freelance projects. So we don't just teach
it once at the end — we **build the habit daily** so that by Day 45 each student has a real audience,
a niche, a portfolio, and a pipeline — not just a certificate.

## How it runs each day
- **30 minutes**, delivered over a **slide deck** (one per day), same format as the Day-1
  *Software Engineering Basics* deck: a single self-contained `index.html`, Softpro brand, keyboard
  navigation, speaker notes in the folder `README.md`.
- Suggested slot: the **first 30 min of the session** ("set the intention, then build"), or the
  last 30 min ("reflect + ship today's proof"). Trainer's choice — keep it consistent.
- Every day ends with **one small Career Action** (≤15 min of homework) that *compounds*. Most days
  that action is "ship one piece of content / proof." 45 small actions = a portfolio + an audience.

## Folder layout (lives *inside* each code Day folder)
Each day's career deck sits in a `career-talk/` subfolder of that day's code folder, so everything
for a given day travels together:
```
Phase-1-Foundations/Day-01-Python-Setup-…/
├── README.md                     (the code day)
├── 00-…/ 01-…/ … exercises/      (code modules)
└── career-talk/
    ├── index.html                The 30-min deck (self-contained)
    └── README.md                 Speaker notes + today's Career Action
```
Decks reuse a shared **template** (same theme variables, nav, progress bar) so they look consistent
and are fast to produce. ~14–18 slides per deck ≈ 30 min with discussion.

> Note: career Days 1–45 don't 1:1 match code-day topics, but they share the day *number*. The
> `career-talk/` folder is independent of that day's code modules.

## The 45-day arc (5 themes, aligned to the 4 course phases)

| Theme | Days | Goal |
|-------|:----:|------|
| 1 — Mindset & Setup | 1–7 | Understand the real game; set up every profile |
| 2 — Build in Public | 8–15 | Start shipping content; ride Project 1's launch |
| 3 — Specialization & Proof | 16–25 | Pick a niche; turn projects into case studies |
| 4 — Routes to Income | 26–37 | Freelance, jobs, outreach, open source, network |
| 5 — Conversion & Launch | 38–45 | Resume, interview, offer, portfolio go-live |

The track is **synced to the code**: when students ship Project 1/2/3 and the capstone, this track
turns each into content and proof. Days 39/44/45 deliberately reinforce the technical Phase-4
sessions (LinkedIn, resume, portfolio launch) rather than repeat them.

## What every student walks out with (this track's deliverables)
- A **set up + optimized presence**: LinkedIn, GitHub profile, (optional) X/YouTube/blog
- A **defined niche** + positioning statement
- **~30+ pieces of published content** (built in public over 45 days)
- **3 project case studies** + a flagship demo
- A **freelance profile** + first outreach sent, and/or **tailored applications** out
- **Resume + STAR stories + portfolio site**, ready to convert
- A **90-day post-course plan** so momentum doesn't die at Day 45

## Production notes (for whoever builds the decks)
- One `career-talk/` folder inside each day's code folder. Reuse the shared deck template (theme variables, nav, progress bar) from the Day-1 career deck.
- ~16–20 slides each. Speaker notes table in each day's `README.md` (slide # → talking point), same style as the Day-1 deck README.
- Keep examples **India-context** and **free-tier** (₹ rates, Indian platforms/communities where relevant), matching the cohort.
- Cross-link: each deck's last slide → that day's code modules; each code day can link back to its career deck.
